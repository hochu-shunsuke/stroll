# 作業メモ

シードから生成される自然を歩く世界。全体像と拡張の判断基準は `README.md` にある。

`.claude/` があれば、**コードを触る前に読むこと**。方向性の判断と、
一度やって取り消したことの記録が入っている（git には入れていない）。
同じ提案を繰り返さないために置いてある。

## 増やすかどうかの判断

> **`(x, z)` とシードだけから、純粋な関数として計算できるか？**

これを満たす限り、無限生成も URL 共有も壊れない。満たさないもの（川の流れなど、
大域の情報が要るもの）は、入れる前に設計ごと考え直すこと。

`heightAt()` は 1 チャンクあたり約 9,400 回呼ばれる。層を足すほど生成が重くなる。

## 世界の作られ方（地形・気候・宝物）

3 つの層が重なって景色ができる。すべて `(x,z)` とシードの純粋関数。

1. **標高** `terrain.heightAt()` — 大陸→侵食→尾根→丘→細部→湖の 6 層ノイズ。
   洞窟は作らない（高さマップなので原理的に出ない）。
2. **気候** 気温 × 湿り気で気候帯（砂漠・草原・森・密林・寒帯・雪）。
   気温は標高で下がる（高い山の頂は暑い地方でも雪）。色は `terrain.shade()`、
   植生は `world/scatter.ts` が気候で出し分ける。**標高には触っていない。**
3. **宝物** `world/special.ts` — まれに現れ、気候を無視して上書きするレア区画。
   秋・桜・青草原など。追加は `SPECIAL_BIOMES` に `forest()`/`meadow()` で 1 行。
   全く別の宝物（霧など）は生の `SpecialBiome` を書く。型は平たいまま残してある。
   植生の種類番号は `world/vegetationKinds.ts`（循環を避ける中立点）。

宝物を足す手順: ①`SPECIAL_BIOMES` に 1 行 ②木が要るなら `vegetationKinds` に
KIND を足し `render/vegetation.ts` に形を足す。3 ファイルを番号で結ぶ。

## 静かに壊れる場所

型検査を通るのに壊れる。触るときはここを思い出すこと。

- **`terrain.heightOnGrid()` と `chunk.buildChunkArrays()` は同じ三角形の切り方をしている前提。**
  片方だけ変えると足元が地面から浮く。機構では強制できていない
- **`LOD_STEPS` と `LOD_RINGS` は同じ長さ。** 起動時に検査している
- **Worker の生成は文字列リテラル**（`new URL('../world/worker.ts', import.meta.url)`）。
  ファイルを移動しても型検査は気づかず、ビルドで初めて落ちる
- **`EYE_HEIGHT` は通信で送る `y` の意味そのもの。** アバターの身長もこれに従う。
  値を変えると他人のアバターだけ地面に沈む
- **透明なものの描画順は `render/order.ts` に集約。** 水面はカメラ追従の板なので
  three の並べ替えでは常に「一番近い」。新しい透明物を足すと水の下敷きになる
- **本番の中継先は同一オリジンから決めており、環境変数を見ない。**
  `.env.local` はモードに関係なく本番ビルドでも読まれるため（一度これで壊した）
- **`world/special.ts` の宝物抽選はシードを混ぜる（`seedSalt`）。** 忘れると
  全世界で宝物の位置が同じになる（一度これで壊した）
- **非インデックスのジオメトリを頂点番号で変形しない。** 同じ角が複数頂点として
  重複しているので、番号で乱数を引くと面が裂ける。座標から引くこと（岩で踏んだ）
- **再接続の死体（幽霊アバター）対策は 2 段構え。** クライアントは cid
  （`sessionStorage` の固定 ID）を再接続・再読み込みでも同じにして送る。
  サーバは同じ cid の古い接続を閉じ、welcome からも除外する。
  cid をメモリだけに持つとスマホの再読み込みで変わり、死体が残る（踏んだ）。
  **未対処の穴**: 誰も再接続してこない死体（タブを閉じたまま close も届かない）は
  消す人がいない。実際に観測したら、サーバに「最後に見た時刻」を持たせて
  古い接続を掃除する（alarm か、通信時に間引く）。今は先回りしない。

## 合言葉と URL

- 合言葉は英数字 **8 文字固定**。決まりは `shared/seed.ts` にだけ書き、
  クライアントとサーバの両方が使う。**片方にだけ検証を足さないこと。**
- URL は `https://.../#k7p2mq9x`。合言葉は **`#`（ハッシュ）**に載る。
  `#` 以降はサーバに送られないので、打ち間違えても 404 が起きず、
  サーバの経路分けも要らない。クエリやパスに戻すとこの利点が消える。
- WebSocket だけは `/ws?seed=...` で内部的に合言葉を渡す（利用者は見ない）。

## 足さないもの

- **洞窟・穴・閉塞的なもの。** 高さマップなので原理的に発生しない。そのまま保つ
- **外部アセット。** 地形も植生も音も、その場で計算して作る。
  シードだけで世界が再現できることが URL 共有の前提になっている
- **目的・期限・競争。** 「目的も期限もない」ことが売り。義務を作る要素は価値を削る

## コマンド

```bash
npm run dev      # クライアント（vite。LAN 公開されるのでスマホからも見られる）
npm run relay    # 中継サーバ（別ターミナル。wrangler dev）
npm run ci       # 型(client/server)＋ビルド＋seed テスト。push 前にこれを通す
npm test         # seed の単体テストだけ
npm run deploy   # ビルドして Cloudflare へ。本体と中継が同時に出る
```

push / PR では GitHub Actions（`.github/workflows/ci.yml`）が `npm run ci` を回す。
中継サーバ相手のプロトコル確認はネットワーク依存なので CI に入れていない
（`scripts/`相当は無し。手元で `npm run relay` を立てて別途叩く）。

## branch を切って進めた方がよい変更

見た目や地形を大きく触るときは branch を切り、まず `npm run dev` の
LAN URL でスマホ確認 → 良ければ `npm run ci` を通して `--no-ff` で main へ。
プレビューを公開 URL で見たいときは Cloudflare の Preview URLs を一度
ダッシュボードで有効化する（未設定なので今は出せない）。
