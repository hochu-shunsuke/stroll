# stroll

シードから生成される、自然を歩くだけの世界。目的も、期限もない。

URL を共有すると、同じ地形を友達と一緒に歩ける。

```
https://stroll.hochu.workers.dev/#k7p2mq9x
                                 ^^^^^^^^ 合言葉。英数字 8 文字
```

公開版: https://stroll.hochu.workers.dev

## 歩き方

キーボード: `WASD` 歩く / `W`二度押し 走る / `Space` 跳ぶ /
`Space`二度押し 飛ぶ（見ている方へ。上昇 `Space`・下降 `Shift`）/ `M` 消音 / `Esc` 一時停止。
スマホ: 画面の左半分をなぞって歩き、右半分をなぞって見回す。

同じ合言葉なら同じ地形。URL を送れば友達と同じ世界を一緒に歩ける。

## 動かす

```bash
npm install
npm run dev      # http://localhost:5173（LAN 公開。スマホからも見られる）
npm run relay    # 別のターミナルで。マルチプレイの中継サーバ
npm run ci       # 型・ビルド・テスト（push 前に通す）
npm run deploy   # ビルドして Cloudflare へ。本体と中継を同時に配る
```

`npm run relay` を動かさなくても 1 人で歩ける。

## 設計の芯

**外部アセットを一切使わない。** 地形も、木も、音も、その場で計算して作る。
だから合言葉（`#...`）さえ同じなら、誰のブラウザでも寸分違わず同じ世界が
再現される。サーバは地形を送らない。送るのは座標だけ。

この 1 点が「無限に広い世界」と「URL を渡すだけで一緒に歩ける」を同時に成立させている。

**洞窟や穴は作らない。** 地形は高さマップなので、原理的に閉じた空間ができない。
怖くない、開けた場所だけの世界にするための選択。

## 世界の三層

歩くと景色が変わるのは、3 つの層が重なっているから。すべて `(x,z)` とシードの純粋関数。

1. **標高** — 平原・丘・山・湖・島。
2. **気候**（気温 × 湿り気）— 砂漠・草原・森・密林・寒帯・雪。歩くと移り変わる。
3. **宝物** — まれに現れ、気候を無視して上書きするレア区画。秋・桜・青い草原など。

## 拡張するときの判断基準

> **`(x, z)` とシードだけから、純粋な関数として計算できるか？**

これを満たす限り、無限生成も URL 共有も壊れない。

| やりたいこと | 難易度 | どこを触るか |
|---|---|---|
| 新しい気候帯 | 易 | `world/terrain.ts` の `shade()` の色 ramp と `scatter.ts` の植生条件 |
| 新しい宝物（色の森・草原） | 易 | `world/special.ts` に `forest()`/`meadow()` を 1 行 |
| 新しい木・岩・草 | 易 | `vegetationKinds.ts` に KIND、`render/vegetation.ts` に形、`scatter.ts` に配置 |
| 村・遺跡 | 中 | `heightAt()` が村の中心付近で平坦な台地へブレンドする |
| 川・滝 | 難 | 水は「どちらへ流れるか」という大域情報が要る。局所計算と相性が悪い |

`heightAt()` は 1 チャンクあたり約 9,400 回呼ばれる。層を足すほど生成が重くなるので、
その予算内で考えること。

## 構成

依存は一方向に流れる（循環なし）。逆向きの import を足さないこと。
`main.ts` だけが全部を知り、組み立てる。

```
core ← world ← player ← ui
core ← world ← render → (net, player)
core ← audio
net（何にも依存しない）      shared（合言葉の決まり。client と server が使う）
```

- `render` は世界・入力・通信を束ねて描く（一番多くを知る層）。
- `ui` は入力方式の都合で `player` に触れる（`touch.ts` が操作を渡す）。

| ディレクトリ | 役割 |
|---|---|
| `core/` | シード付き乱数とハッシュ |
| `shared/` | クライアントとサーバで共有する決まり（合言葉の検証） |
| `world/` | 地形・気候・宝物の定義、チャンクのメッシュ化、植生の配置。描画を知らない |
| `render/` | 空・水・植生・アバター・チャンクの出し入れ |
| `player/` | 入力と移動 |
| `audio/` | 足音と環境音。すべて合成 |
| `net/` | 部屋への接続 |
| `ui/` | 開始画面・合言葉入力・タッチ操作 |
| `server/` | Cloudflare Worker + Durable Object の中継 |

## 触るときの注意（型検査を通るのに壊れる場所）

- `terrain.heightOnGrid()` と `chunk.buildChunkArrays()` は
  **同じ三角形の切り方**をしている前提。片方だけ変えると足元が地面から浮く。
- `chunk.ts` の `LOD_STEPS` と `LOD_RINGS` は同じ長さであること（起動時に検査している）。
- Worker の生成は `new URL('../world/worker.ts', import.meta.url)` という
  **文字列リテラル**。ファイルを移動しても型検査では気づけない。
- `world/special.ts` の宝物抽選はシードを混ぜる。忘れると全世界で宝物が同じ位置に出る。
- 非インデックスのジオメトリを頂点番号で変形しない（同じ角が裂ける）。座標から乱数を引く。
