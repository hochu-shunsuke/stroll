# stroll

自動生成された自然を歩くだけの世界。目的も、期限もない。

URL を共有すると、同じ地形の同じ場所を友達と一緒に歩ける。

公開版: https://stroll-relay.hochu.workers.dev

## 動かす

```bash
npm install
npm run dev      # http://localhost:5173
npm run relay    # 別のターミナルで。マルチプレイの中継サーバ
```

`npm run relay` を動かさなくても 1 人で歩ける。`.env.local` の
`VITE_RELAY_URL` を消せば通信そのものを行わない。

```bash
npm run deploy   # ビルドして Cloudflare へ。ゲーム本体と中継を同時に配る
```

## 数える

https://stroll-relay.hochu.workers.dev/stats

記録するのは 4 つの合計値だけ。Cookie も識別子も IP も保存しない。
だから同意バナーが要らない（静かに歩く体験の入口にバナーを挟みたくない）。

見るべきは PV ではなくこの 1 点：

> **共有された URL を受け取った人が、実際に歩き始めたか**

「一緒に歩ける」が売りである以上、1 人が何人連れてくるかが全て。
`連れてきた人数` が 1 を超えないうちは、広告を打っても穴の空いたバケツになる。

数え直したいときは `server/index.ts` の `STATS_KEY` を変える。
別の入れ物が作られ、全てゼロから始まる。

## 設計の芯

**外部アセットを一切使わない。** 地形も、木も、音も、その場で計算して作る。
だからシード（`?seed=...`）さえ同じなら、誰のブラウザでも寸分違わず同じ世界が
再現される。サーバは地形を送らない。送るのは座標だけ。

この 1 点が「無限に広い世界」と「URL を渡すだけで一緒に歩ける」を同時に成立させている。

**洞窟や穴は作らない。** 地形は高さマップなので、原理的に閉じた空間ができない。
怖くない、開けた場所だけの世界にするための選択。

## 拡張するときの判断基準

> **`(x, z)` とシードだけから、純粋な関数として計算できるか？**

これを満たす限り、無限生成も URL 共有も壊れない。

| やりたいこと | 難易度 | どこを触るか |
|---|---|---|
| 新しいバイオーム | 易 | `world/terrain.ts` の `shade()` に色帯を追加 |
| 新しい木・岩・草 | 易 | `world/scatter.ts` の `SPECS` に 1 要素、`render/vegetation.ts` に形 |
| 村・遺跡 | 中 | `heightAt()` が村の中心付近で平坦な台地へブレンドする |
| 川・滝 | 難 | 水は「どちらへ流れるか」という大域情報が要る。局所計算と相性が悪い |

`heightAt()` は 1 チャンクあたり約 9,400 回呼ばれる。層を足すほど生成が重くなるので、
その予算内で考えること。

## 構成

依存は一方向に流れる。逆向きの import を足さないこと。

```
core   ←  world  ←  render
              ↖ player
core   ←  audio          ui, net（独立）
                    main.ts → 全部
```

| ディレクトリ | 役割 |
|---|---|
| `core/` | シード付き乱数とハッシュ |
| `world/` | 地形の定義、チャンクのメッシュ化、植生の配置。描画を知らない |
| `render/` | 空・水・植生・アバター・チャンクの出し入れ |
| `player/` | 入力と移動 |
| `audio/` | 足音と環境音。すべて合成 |
| `net/` | 部屋への接続 |
| `ui/` | 開始画面と最小限の表示 |
| `server/` | Cloudflare Worker + Durable Object の中継 |

## 触るときの注意

- `terrain.heightOnGrid()` と `chunk.buildChunkArrays()` は
  **同じ三角形の切り方**をしている前提。片方だけ変えると足元が地面から浮く。
- `chunk.ts` の `LOD_STEPS` と `LOD_RINGS` は同じ長さであること（起動時に検査している）。
- Worker の生成は `new URL('../world/worker.ts', import.meta.url)` という
  **文字列リテラル**。ファイルを移動しても型検査では気づけない。
